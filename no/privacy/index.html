Personvernerklæring

1) Behandlingsansvarlig

Behandlingsansvarlig for personopplysninger er:
Canario-Lab / Chrumek App
Kontakt-e-post: chrumek@chrumek.app

2) Omfang og formål

Chrumek-appen hjelper deg med å følge sparing ved å registrere «fristelser» og bekreftede overføringer / beløp som er satt til side.

Vi driver ikke brukerkontoer og samler ikke inn data som lar oss identifisere deg direkte.

3) Data som behandles i appen (lokalt på enheten din)

Data du legger inn i appen (for eksempel spareposter, beløp, datoer, kategorier, mål og innstillinger) lagres lokalt på enheten din.

Det sendes ikke automatisk til oss eller til våre servere.

4) Funksjoner som krever systemtilgang

Appen kan bruke systemfunksjoner og relaterte data bare i den grad det er nødvendig for at den skal fungere:

  • Varsler (UNUserNotificationCenter) - hvis du aktiverer påminnelser, leverer iOS varsler i samsvar med innstillingene dine.
  • Språk / region / valuta / tema - brukes til å formatere appens innhold og utseende (uten å sende noe til oss).

5) Feilrapporter / kontakt med support

Hvis du bruker alternativet «Rapporter en feil» eller kontakter support:

  • kan appen opprette et utkast til e-post i e-postappen din,
  • kan innholdet inneholde diagnostisk informasjon (for eksempel appversjon, iOS-versjon eller språk- / valutainnstillinger) - akkurat det du ser i meldingen før du sender den,
  • sendes e-posten bare hvis du selv godkjenner det.

En sendt e-post behandles av e-postleverandøren din og av oss som mottaker, utelukkende for å håndtere forespørselen din.

6) Kjøp og abonnementer (Apple + RevenueCat)

Hvis betalte funksjoner (abonnementer) er tilgjengelige i appen:

  • behandles betalinger av Apple (App Store / In-App Purchases / StoreKit),
  • bruker appen RevenueCat som infrastrukturen for abonnementer (blant annet for å vise tilbud, verifisere abonnementsstatus og gi Premium-tilgang).

Som en del av abonnementshåndteringen kan tekniske kjøpsrelaterte data behandles, for eksempel:

  • transaksjons- / kjøpsidentifikatorer og produktinformasjon,
  • abonnementsstatus (aktiv / inaktiv, fornyelser),
  • en teknisk brukeridentifikator som brukes for å knytte abonnementsstatusen sammen (for eksempel en anonym App User ID generert av SDK-en).

RevenueCat behandler disse dataene for å levere tjenester for abonnementshåndtering (som databehandler), og Apple behandler betalingsrelaterte data i samsvar med sine egne regler.

7) Analyse og sporing

Appen bruker ikke analyseverktøy eller annonse-SDK-er for å spore brukere og selger ikke data.

8) Nettsted (landingsside)

Hvis du besøker nettstedet vårt, kan hostingleverandøren (for eksempel GitHub Pages) automatisk behandle standard tekniske data i serverlogger (inkludert IP-adresse, nettlesertype og dato og klokkeslett for forespørselen) for sikkerhet, diagnostikk og tjenestekontinuitet. Omfanget og reglene for slik behandling følger hostingleverandørens retningslinjer.

9) Datamottakere

Data kan bare utleveres til:

  • leverandører av betaling og abonnement: Apple og RevenueCat (for abonnementshåndtering),
  • offentlige myndigheter, hvis loven krever det.

10) Lagringsperiode

  • Data som legges inn i appen lagres lokalt til du fjerner dem (for eksempel ved å slette poster eller slette appen).
  • E-postkorrespondanse lagres så lenge det er nødvendig for å håndtere forespørselen og forsvare seg mot krav (ved behov).

11) Brukerrettigheter (EU / EØS)

Hvis vi i et konkret tilfelle behandler personopplysningene dine (for eksempel når du sender oss en e-post), har du rett til:

  • innsyn, retting, sletting eller begrensning av behandling,
  • å protestere,
  • å klage til den kompetente tilsynsmyndigheten.

Omfanget av informasjonsforpliktelsene følger særlig av artikkel 13 i GDPR.

12) Sikkerhet

Vi bruker rimelige organisatoriske og tekniske tiltak som passer til tjenestens natur. Siden appdata lagres lokalt, er det også en viktig del av sikkerheten å beskytte enheten din (kode / Face ID).

13) Endringer i erklæringen

Denne erklæringen kan bli oppdatert når appfunksjoner eller lovkrav endrer seg. Den gjeldende versjonen blir publisert på appens nettsted.
