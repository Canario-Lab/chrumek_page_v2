隐私政策

1) 数据控制者

个人数据控制者为：
Canario-Lab / Chrumek App
联系邮箱：chrumek@chrumek.app

2) 范围与目的

Chrumek 是一款用来追踪储蓄的应用，通过记录“诱惑”以及确认后的转账/实际存下的金额来帮助你建立储蓄习惯。

我们不提供用户账户系统，也不收集能够直接识别你身份的数据。

3) 应用内处理的数据（仅存储在设备本地）

你在应用中输入的数据（例如储蓄记录、金额、日期、分类、目标、设置）都保存在你的设备本地。

这些数据不会自动发送给我们，也不会发送到我们的服务器。

4) 需要系统权限的功能

应用只会在运行所必需的范围内使用系统功能及相关数据：

  • 通知（UNUserNotificationCenter） - 如果你启用了提醒，iOS 会按照你的设置发送通知。
  • 语言 / 地区 / 货币 / 主题 - 仅用于格式化应用内容和显示效果，不会发送给我们。

5) 错误报告 / 联系支持

如果你使用“报告错误”功能或联系支持：

  • 应用可能会在你的邮件应用中创建一封邮件草稿，
  • 草稿内容可能包含诊断信息（例如应用版本、iOS 版本、语言/货币设置）——也就是你在发送前能看到的那些内容，
  • 只有在你自己确认后，邮件才会发送。
    已发送的邮件会由你的邮件服务提供商以及我们（作为收件人）处理，仅用于处理你的请求。

6) 购买与 Premium 访问（Apple + RevenueCat）

如果应用提供付费功能（订阅或一次性购买终身访问权）：

  • 付款由 Apple 处理（App Store / In-App Purchases / StoreKit），
  • 应用使用 RevenueCat 作为购买与 Premium 访问处理的基础设施提供商（包括展示优惠、验证访问状态、识别有效订阅或终身访问权购买，以及授予 Premium 访问权限）。
    在处理购买与 Premium 访问时，可能会涉及与购买相关的技术数据，例如：
  • 交易/购买标识符以及商品信息，
  • Premium 访问状态（有效/无效）、订阅状态（例如续订情况）或一次性购买的信息，
  • 用于关联 Premium 访问状态的技术用户标识符（例如由 SDK 生成的匿名 App User ID）。
    RevenueCat 处理这些数据以提供购买与 Premium 访问处理服务，Apple 则按照自己的政策处理与付款相关的数据。

7) 分析与跟踪

应用可能会使用 Firebase Analytics 收集关于应用使用方式的匿名数据（例如添加记录、完成转存、达到免费转存次数上限、显示 Premium 优惠页面，或点击该页面的主按钮）。

这些数据不包含记录内容、金额、目标名称、类别、价格、购买方案、支付数据或可直接识别你的信息。数据不会与用户身份或某一具体设备关联。我们仅将这些数据用于分析应用运行情况、提升稳定性和改进功能。

我们不会将分析数据用于出售数据或广告画像。

8) 网站

如果你访问我们的网站，托管服务提供商（例如 GitHub Pages）可能会在服务器日志中自动处理标准技术数据（包括 IP 地址、浏览器类型、请求日期和时间），用于安全、诊断和服务连续性。处理范围和规则取决于托管服务提供商的政策。

9) 数据接收方

数据只可能披露给：

  • 支付和 Premium 访问管理服务提供方：Apple 和 RevenueCat（处理购买和访问状态），
  • 分析服务提供方：Google Firebase Analytics（应用事件的匿名分析），
  • 法律要求时的公共机构。

10) 保存期限

  • 你在应用中输入的数据会一直保存在本地，直到你自行删除（例如删除记录或删除应用）。
  • 电子邮件往来数据会在处理请求及必要时用于抗辩索赔所需的期间内保存。

11) 用户权利（欧盟 / 欧洲经济区）

如果在特定情况下我们处理了你的个人数据（例如你给我们发邮件时），你有权：

  • 访问、更正、删除及限制处理，
  • 提出异议，
  • 向有管辖权的监管机构投诉。
    相关信息义务的范围，尤其依据 GDPR 第 13 条。

12) 安全

我们会采取与服务性质相称的合理组织和技术措施。由于应用数据保存在本地，保护你的设备（密码 / Face ID）同样是安全的重要组成部分。

13) 政策变更

随着应用功能或法律要求的变化，本政策可能会更新。最新版本会发布在应用网站上。
