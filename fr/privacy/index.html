Politique de confidentialité

1) Responsable du traitement

Le responsable du traitement des données personnelles est :
Canario-Lab / Chrumek App
E-mail de contact : chrumek@chrumek.app

2) Portée et finalité

L’application Chrumek aide à suivre l’épargne en enregistrant des « tentations » ainsi que des transferts / montants mis de côté et confirmés.

Nous ne gérons pas de comptes utilisateurs et nous ne collectons pas de données permettant de vous identifier directement.

3) Données traitées dans l’application (localement sur votre appareil)

Les données que vous saisissez dans l’application (par exemple les entrées d’épargne, montants, dates, catégories, objectifs et réglages) sont stockées localement sur votre appareil.

Elles ne sont pas envoyées automatiquement vers nous ni vers nos serveurs.

4) Fonctions nécessitant un accès système

L’application peut utiliser des fonctions système et les données associées uniquement dans la mesure nécessaire à son fonctionnement :

  • Notifications (UNUserNotificationCenter) - si vous activez les rappels, iOS envoie les notifications selon vos réglages.
  • Langue / région / devise / thème - utilisés pour formater le contenu et l’apparence de l’application (sans rien nous envoyer).

5) Signalement d’erreurs / contact avec le support

Si vous utilisez l’option « Signaler un bug » ou contactez le support :

  • l’application peut créer un brouillon d’e-mail dans votre application de messagerie,
  • le contenu peut inclure des informations de diagnostic (par exemple la version de l’application, la version d’iOS ou les réglages de langue / devise) - exactement ce que vous voyez dans le message avant de l’envoyer,
  • l’e-mail n’est envoyé que si vous l’approuvez vous-même.

Un e-mail envoyé est traité par votre fournisseur de messagerie ainsi que par nous en tant que destinataire, uniquement afin de traiter votre demande.

6) Achats et abonnements (Apple + RevenueCat)

Si des fonctions payantes (abonnements) sont disponibles dans l’application :

  • les paiements sont gérés par Apple (App Store / In-App Purchases / StoreKit),
  • l’application utilise RevenueCat comme prestataire d’infrastructure pour les abonnements (notamment pour présenter les offres, vérifier le statut de l’abonnement et accorder l’accès premium).

Dans le cadre du traitement des abonnements, des données techniques liées à l’achat peuvent être traitées, par exemple :

  • des identifiants de transaction / d’achat et des informations sur le produit,
  • le statut de l’abonnement (actif / inactif, renouvellements),
  • un identifiant technique d’utilisateur utilisé pour rattacher le statut de l’abonnement (par exemple un App User ID anonyme généré par le SDK).

RevenueCat traite ces données afin de fournir le service de gestion des abonnements (en tant que sous-traitant), et Apple traite les données liées au paiement selon ses propres règles.

7) Analytique et suivi

L’application n’utilise pas d’outils d’analytics ni de SDK publicitaires pour suivre les utilisateurs et ne vend pas de données.

8) Site web (landing page)

Si vous visitez notre site web, le prestataire d’hébergement (par exemple GitHub Pages) peut traiter automatiquement des données techniques standard dans les journaux du serveur (notamment l’adresse IP, le type de navigateur ainsi que la date et l’heure de la requête) à des fins de sécurité, de diagnostic et de continuité du service. L’étendue et les règles de ce traitement suivent les politiques du prestataire d’hébergement.

9) Destinataires des données

Les données ne peuvent être communiquées qu’aux destinataires suivants :

  • les prestataires de paiement et d’abonnement : Apple et RevenueCat (pour la gestion des abonnements),
  • les autorités publiques, si la loi l’exige.

10) Durée de conservation

  • Les données saisies dans l’application sont conservées localement jusqu’à ce que vous les supprimiez (par exemple en supprimant des entrées ou en supprimant l’application).
  • La correspondance par e-mail est conservée aussi longtemps que nécessaire pour traiter la demande et, si besoin, se défendre contre d’éventuelles réclamations.

11) Droits de l’utilisateur (UE / EEE)

Si, dans un cas donné, nous traitons vos données personnelles (par exemple lorsque vous nous envoyez un e-mail), vous avez le droit :

  • d’accéder à vos données, de les rectifier, de les effacer ou d’en limiter le traitement,
  • de vous opposer au traitement,
  • de déposer une réclamation auprès de l’autorité de contrôle compétente.

L’étendue des obligations d’information résulte notamment de l’article 13 du RGPD.

12) Sécurité

Nous appliquons des mesures organisationnelles et techniques raisonnables, adaptées à la nature du service. Comme les données de l’application sont stockées localement, la protection de votre appareil (code, Face ID) fait également partie de la sécurité.

13) Modifications de la politique

Cette politique peut être mise à jour si les fonctions de l’application ou les exigences légales évoluent. La version en vigueur sera publiée sur le site de l’application.
